# 🇮🇳 Project Overview

This project presents an **India District-Level Socioeconomic Digital Twin**, an AI-powered analytical framework that integrates multiple official Government of India datasets into a unified system for measuring, analyzing, visualizing, explaining, and predicting household socioeconomic wellbeing.

The Digital Twin combines large-scale government microdata, statistical analytics, GIS, spatial econometrics, machine learning, explainable AI, and interactive dashboards to provide a multidimensional representation of India's socioeconomic landscape.

Unlike traditional socioeconomic studies that rely on a single indicator such as income or expenditure, this framework jointly models household wellbeing through multiple interconnected dimensions including food security, welfare access, asset ownership, vulnerability, inflation, and geographic context.

The project demonstrates how official government data can be transformed into an intelligent decision-support system capable of assisting policymakers, researchers, and planners.

# 📒 Notebook 1 — Data Engineering & Feature Engineering

The first notebook establishes the foundation of the **India District-Level Socioeconomic Digital Twin** by transforming multiple raw Government of India datasets into a unified household-level analytical database.

This stage focuses on data acquisition, cleaning, integration, feature engineering, inflation adjustment, and preparation of the final dataset used for statistical analysis, GIS, and machine learning.

---

## 🎯 Objectives

- Integrate multiple Government datasets into a unified analytical framework.
- Create a unique household identifier across survey levels.
- Clean and standardize raw survey records.
- Engineer meaningful socioeconomic indicators.
- Adjust monetary values using Consumer Price Index (CPI).
- Produce a high-quality analytical dataset for downstream modeling.

---

# 📦 Step 1 — Data Collection

Five official Government datasets were integrated into this project.

| Dataset | Description |
|----------|-------------|
| MOSPI HCES Level-5 | Household Food Consumption |
| MOSPI HCES Level-9 | Welfare Expenditure |
| MOSPI HCES Level-13 | Household Durable Assets |
| Consumer Price Index (CPI) | Inflation Adjustment |
| GADM Administrative Boundaries | State & District GIS Layers |

---

# 🧹 Step 2 — Data Cleaning

Extensive preprocessing was performed to ensure data quality.

### Major preprocessing tasks

- Missing value analysis
- Duplicate record removal
- Data type conversion
- Invalid code handling
- Standardization of administrative names
- Survey variable harmonization
- Consistency checking across survey levels

The cleaning process significantly improved the reliability of the integrated analytical dataset.

---

# 🔗 Step 3 — Household Data Integration

Since HCES survey levels are stored independently, household-level integration was required.

A composite household identifier was created using:

- FSU Serial Number
- Sample Household Number

This enabled accurate merging of:

- Food Consumption
- Welfare Expenditure
- Household Assets

into a single household-level dataset.
